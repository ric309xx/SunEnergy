# 冠得科技 網站前端修改紀錄

本文件用於記錄針對 冠得科技 專案（版本：2026-03-02）所進行的各項前端樣式與內容調整，方便未來追蹤與維護。



## 📅 2026 年 3 月 4 日 修改項目

### 1. 實績案例排版與功能升級
- **支援多圖相簿 (Gallery)**：修改 CMS 設定檔 `config.yml`，讓實績案例支援上傳多張次要照片，並於前台顯示為左側縮圖列，點擊即可切換主圖。
- **介面重構**：將案例彈窗 (`#case-modal`) 改為經典的「左圖右數據」對切佈局，避免原本照片過度滿版的壓迫感。
- **年發電量自動估算**：新增 JavaScript 即時運算，透過建置容量自動乘上 1100 小時，產生「預估年發電量 (度)」橘黃色顯眼卡片。
- **影片版面優化**：取消 YouTube 影片的 100% 滿版設計，改為適當比例置中，並加上「相關影片」標題。

### 2. 全站視覺重構與架構升級
- **導入 Tailwind CSS 科技黑金風格**：屏棄過往繁雜的手寫 CSS，全面導入 Tailwind CSS 建立一體化的頂級暗黑科技風格 (`indexref.html` 版型)。
- **建立簡化的 6 步驟流程**：依據需求，將「服務流程」大幅精簡為「需求洽詢、現場勘查、規劃設計、工程建置、併網發電、維運監測」6 個直覺、無壓力的步驟。
- **全站捲動動畫 (Scroll Reveal)**：使用 Intersection Observer 替各大區塊設計平滑淡入動畫，優化網站的高級感與互動性。

### 3. 首頁版面與佈局微調
- **Hero 影片亮度調整**：將首頁背景影片的不透明度提升至 80%，確保背景具備充分的視覺張力，同時維持前方導覽文字的清晰度。
- **服務項目排版優化**：捨棄原先置中的版型，將 Icon 與標題改為同軸（同一水平線）靠左對齊，使得卡片有更佳的科技流線感。
- **標題版面排版統一**：針對「實績案例」與「最新消息」區段，將右上方的灰字說明移至與左側主標題在同一水平線並排排列 (`flex flex-row items-end`)。
- **區塊顯示順序重構**：調整首頁中後段的 HTML DOM 結構順序為「實績案例 ➔ 最新消息 ➔ 服務流程 ➔ 免費諮詢」，同時清理了程式內多餘重複的最新消息標籤區塊。
- **解決本機 JSON 跨域阻擋問題**：由於各大瀏覽器安全限制（CORS），若雙擊開啟會導致讀不到 JSON 資料的問題。現已於 `fetch()` 加入 Catch 異常防呆機制，不再卡住轉圈，而是顯示紅色的清楚錯誤提示。

### 4. 案例與新聞互動優化
- **JSON 資料排序修正**：解決前端與 CMS 管理後台順序顛倒的問題，移除預設的 `.reverse()` 反轉機制，達成所見即所得（後台在最上方排列的，即為前台首頁最左側）。
- **實績案例版面微調**：將彈窗 (Modal) 的左側照片與右側文字的版寬調整為 1:1 (各 50%)，並修正說明文字過長被截斷的問題，改為完整多行展開 (`leading-relaxed`)。
- **最新消息彈窗實作**：為原先無法點擊的新聞卡片，全新實作並套用與案例相同設計語彙之專屬深黑質感 Modal 視窗，提供完整閱讀體驗。

### 5. 視覺回饋與 RWD 尺寸微調 (客戶反饋)
- **首頁主視覺微調**：調降 Hero 區塊手機版的文字層級至適中大小 (`text-[2.5rem]`)，避免畫面過度擁擠。
- **邊緣漸層融合特效**：「關於我們」區塊相片疊加左右/下方過渡漸層，無縫融入科技黑背景，提升視覺沉浸感。
- **響應式對齊與流程標籤**：實績與新聞區塊的標題於手機版自動置中；並為服務流程加入亮黃色的 1~6 步數字徽章，統一修正「工程建置」遺失的圖標。
- **彈窗照片透亮度修正**：大幅限縮 Modal 內左側相片的過暗漸層範圍（僅保留底部與右側 25% 用作轉場過渡），恢復案例實際照片的質感。
- **實績服務點擊小圖還原**：應客戶要求，把首頁「太陽能/風力/儲能」三項圖示恢復為指定的原始插畫圖片，並輔以高質感發光、互動浮升特效。

## 📅 2026 年 3 月 2 日 / 3 月 3 日 修改項目

### 1. 首頁「服務項目 (Services)」區塊優化
- **Icon 圖片去背融合**：為解決具有白色方塊底色的 Icon 圖片與深色卡片背景（`#334155`）不搭的問題。
  - 為 `.service-icon` 容器新增了白色圓形底色（`border-radius: 50%`、`background-color: var(--white)`）。
  - 對內部的 `img` 應用了 `mix-blend-mode: multiply` 屬性，成功去除原始圖片的白底白邊，使其完美融合。
- **Icon 尺寸放大**：將原先的 `img` 高度由 `70px` 增至 `85px`，提升視覺衝擊力。
- **解決 Hover 文字反白問題**：
  - 修正在將滑鼠移至卡片時（卡片背景變白）導致原先淺色文字無法閱讀的問題。
  - 新增 Hover 狀態下的文字顏色切換：標題（`h3`）轉為深藍色（`var(--primary-dark)`），副標題段落（`p`）轉為深灰色（`var(--bg-main)`）。
  - 全面添加 `transition` 平滑過渡動畫，提升使用者體驗。

### 2. 首頁「Hero 頂部橫幅」區塊優化
- **替換本地背景影片**：
  - 將原本位於 `index.html` 中的線上開源影片網址（`https://cdn.pixabay.com/...`），替換為客戶提供的本地影片檔案 `head.mp4`。
- **調亮影片遮罩透明度 (Overlay)**：
  - 為避免替換後的影片整體過暗，調整了 `.hero-overlay` 的線性漸層設定。
  - 左側深度由原先的 `0.9` 降至 `0.6`。
  - 右側深度由原先的 `0.4` 降至 `0.2`。
  - 修改後既能確保白色標題清楚顯示，也能讓背景影片透出更多光彩。

### 3. 專案版本控制
- 將以上所有更動初始化（`git init`），並完成第一次的 `git commit`。
- Commit 訊息為：*"feat: 更新首頁服務項目 Icon 樣式、增加滑鼠懸停效果及切換首頁影片背景為本地影片並調亮"*

### 4. 介面設計優化 (Premium Apple Aesthetic)
- **按鈕材質與形狀更新**：將 `.btn-primary` 改為實心黑白配色與高級膠囊圓角，`.btn-secondary` 替換為帶有 `backdrop-filter` 模糊邊界的無框毛玻璃效果。
- **側邊浮動欄升級**：將右側 `.floating-sidebar` 轉為深色半透明加上 `blur` 的高級感設計，更符合高階企業品牌形象。

### 5. CMS 介面與前端顯示同步
- **移除前端陣列反向讀取**：因 Decap CMS 預設新增資料於最末端，將 `script.js` 取消 `reverse()` 反轉，改為忠實呈現後台拖曳順序（新 -> 舊）。
- **簡化 Admin 入口介面**：移除預設漸層背景，改用「白底深色字 + 圓角小陰影」的極簡設計，提升專業度。

## 🛠️ 系統技術堆疊與架構 (Technology Stack)

為了讓後續接手的人員或業主能快速了解本網站的運作基礎，以下整理了本專案所使用的核心技術。本網站採用極簡且高效的「無伺服器 (Serverless)」靜態架構，所有設計皆為了「極速載入、高度客製化、免主機維護費」而生：

1. **主流前端技術與 UI 框架**
   - **HTML5**: 負責定義網頁的標準結構與語意。
   - **Tailwind CSS**: 摒棄老舊繁雜的樣式表，導入時下最先進的 Tailwind Utility-First 框架，快速建構高品質的暗黑科技感、毛玻璃特效與動態光暈，並確保 100% 響應式完美呈現在各尺寸裝置。
   - **Vanilla JavaScript (原生 JS)**: 無冗餘庫的純淨動態腳本，搭配原生 Fetch API 將 CMS 的 JSON 料源完美轉接，動態生成 Tailwind 卡片元件。

2. **後台內容管理系統 (Decap CMS)**
   - **Decap CMS (前身 Netlify CMS)**: 輕量級的「無頭式 (Headless) 內容管理系統」。業主只需登入一個專屬網址，就能像發臉書一樣直接上傳圖片、修改文字。
   - **運作模式**：系統沒有傳統資料庫。業主在後台新增文章時，CMS 會直接把這些內容寫出並打包成純粹的 `.json` 檔案。
   - **優點**：架構極度輕快，不需動用工程師干預，且完全沒有資料庫被駭客入侵的風險。

3. **身分驗證與雲端託管 (Netlify)**
   - **Netlify Identity**: 提供後台安全的 Email 登入驗證機制與管理面板。
   - **Netlify Hosting**: 專屬的頂級雲端空間伺服器。當業主在 CMS 按下「發布」後，Netlify 會在幾秒鐘內自動抓取最新圖文，並發布到全球 CDN 節點提供光速下載，同時自帶 SSL (https) 安全加密憑證。

4. **版本控制與最高權限儲存庫 (Git & GitHub)**
   - **GitHub (Git-Gateway)**: 扮演我們專案的「最強防護資料庫」與「時光機」。不論是工程師寫的程式碼，或是業主透過 CMS 發布的每一篇新聞，全部都會轉化為一筆筆的 Git 紀錄保存在安全的主機上，具備無限次安全版本回溯能力。
